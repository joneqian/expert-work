# Memory P5b-2b Implementation Plan — 预测复核 + correction 标记

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Land ⑦ per-fact predictive review (extraction emits `expected_valid_days`; a consolidator sub-pass proactively re-reviews facts whose window is due — still-true → renew/extend, false → `expire()`) and ⑧ extraction-prompt correction marking (the LLM flags "explicit correction of prior info", raising reconcile UPDATE recall — zero extra LLM calls).

**Architecture:** The extraction pipeline gains two per-item signals — `expected_valid_days` (persisted onto `memory_item`, drives the due-scan) and `is_correction` (transient, threaded into reconcile's LLM payload, never persisted). The store gains a `list_due_for_review` due-scan and a `renew_review` setter; the "false" branch reuses the existing `expire()`. The consolidator gains SUB-PASS 3 that reuses the lone-item review skeleton (list → for item → LLM verdict → act), gated by a new tenant-config opt-in.

**Tech Stack:** Python 3.12, SQLAlchemy async + pgvector, pytest + testcontainers (integration needs `DOCKER_HOST`), across three services: `orchestrator` (extraction/reconcile), `expert-work-persistence` (store), `control-plane` (consolidator).

## Global Constraints

- **Store parity + integration tests:** every new store method lands in all three — `base.py` (abstract), `sql.py` (`SqlMemoryStore`), `memory.py` (`InMemoryMemoryStore`) — with a committed SQL integration test (real container, `export DOCKER_HOST=unix:///Users/mac/.docker/run/docker.sock`) AND an in-memory test, plus a mutation-verify (invert the predicate → the SQL test must FAIL; revert → PASS). In-memory-only coverage silently misses SQL-specific bugs (existing SQL fixtures all have NULL bi-temporal columns, so an inverted comparison passes CI unless a due/expired fixture exists).
- **Best-effort extraction/reconcile:** a missing or malformed new field (`expected_valid_days`, `is_correction`) must never drop an otherwise-valid memory — default it (None / False) exactly like the existing `_clamp_score` fallback. Follow the surrounding tolerant-parse idiom.
- **Bi-temporal semantics (do not cross wires):** predictive review's still-true path calls `renew_review` (stamps `last_reviewed_at=now`, updates `expected_valid_days`); the false path calls the existing `expire()` (stamps `expired_at`, no successor — orthogonal to `invalid_at` supersession). Never use `soft_delete` for a fact that became false in the world.
- **Opt-in default:** predictive review is OFF by default — a new tenant-config flag `memory_predictive_review_enabled` defaulting `False`, resolved exactly like the existing `memory_purge_enabled`. It has an LLM cost per due fact; tenants opt in.
- **CI scope:** mypy over `packages services/audit-backup-worker/src services/billing-rollup-job/src services/event-log-archive-job/src services/orchestrator/src services/retention-cleanup-job/src` (+ control-plane); `ruff check` full-repo + `ruff format --check`; pytest includes `tools/eval`. Run the full scope, not single files.
- **flush signature touches control-plane:** any change to `flush_messages_to_memory` or `_reconcile_and_apply` reachable from run-end writeback must run the control-plane memory tests (they exercise the real `run_agent` path), not only orchestrator tests.

Add after `list_review_flagged` (near `sql.py:800`). Note the interval arithmetic uses `func.make_interval` (Postgres `make_interval(years, months, weeks, days, ...)` — days is the 4th positional):

```python
    async def list_due_for_review(
        self,
        *,
        tenant_id: UUID,
        user_id: UUID,
        limit: int,
    ) -> list[MemoryItem]:
        now = datetime.now(UTC)
        base_ts = func.coalesce(MemoryItemRow.last_reviewed_at, MemoryItemRow.created_at)
        due_at = base_ts + func.make_interval(0, 0, 0, MemoryItemRow.expected_valid_days)
        stmt = (
            select(MemoryItemRow)
            .where(
                MemoryItemRow.tenant_id == tenant_id,
                MemoryItemRow.user_id == user_id,
                MemoryItemRow.deleted_at.is_(None),
                MemoryItemRow.invalid_at.is_(None),
                MemoryItemRow.expired_at.is_(None),
                MemoryItemRow.expected_valid_days.isnot(None),
                due_at <= now,
            )

        stmt = (
            update(MemoryItemRow)
            .where(
                MemoryItemRow.id == memory_id,
                MemoryItemRow.tenant_id == tenant_id,
                MemoryItemRow.user_id == user_id,
                MemoryItemRow.deleted_at.is_(None),
            )

        )

            result = await session.execute(stmt)
```

```python
    async def list_due_for_review(
        self,
        *,
        tenant_id: UUID,
        user_id: UUID,
        limit: int,
    ) -> list[MemoryItem]:
        now = datetime.now(UTC)

        def _due_at(row: MemoryItem) -> datetime | None:
            if row.expected_valid_days is None or row.created_at is None:
                return None
            base = row.last_reviewed_at or row.created_at
            return base + timedelta(days=row.expected_valid_days)

        candidates = []
        for row in self._rows:
            if not (
                row.tenant_id == tenant_id
                and row.user_id == user_id
                and row.deleted_at is None
                and row.invalid_at is None
                and row.expired_at is None
                and row.expected_valid_days is not None
            ):
                continue
            due = _due_at(row)
            if due is not None and due <= now:
                candidates.append(row)
        candidates.sort(key=lambda r: _due_at(r) or now)
        return candidates[:limit]

    async def renew_review(
        self,
        *,
        tenant_id: UUID,
        user_id: UUID,
        memory_id: UUID,
        expected_valid_days: int | None,
    ) -> bool:

            if (
                row.id == memory_id
                and row.tenant_id == tenant_id
                and row.user_id == user_id
                and row.deleted_at is None
            ):
                self._rows[idx] = row.model_copy(
                    update={
                        "last_reviewed_at": datetime.now(UTC),
                        "expected_valid_days": expected_valid_days,
                    }
                )
                return True
        return False
```
