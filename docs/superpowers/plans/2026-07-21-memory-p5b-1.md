# Memory P5b-1(溯源 + bi-temporal 核心)Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 给 `memory_item` 加"事实时间有效性"维度(bi-temporal)+ run 级溯源,把 reconcile 的破坏性 UPDATE/DELETE 改成 append-only 版本链,检索默认排除已被取代/已失效的事实。

**Architecture:** 在现有扁平 `memory_item` 行上加 7 列(`source_run_id` + 6 个 bi-temporal 列),不引入 graph。检索排序不变,只在 `retrieve` 的 WHERE 上加"当前有效"过滤。reconcile 的 UPDATE 从原地改写变"关旧行(`invalid_at`+`superseded_by`)开新行(`supersedes`+`valid_at`)",DELETE 从软删变"设 `expired_at`"。失效行保留(可查历史/审计),只是默认检索排除。两个 store 实现(`SqlMemoryStore` + `InMemoryMemoryStore`)全程 parity。

**Tech Stack:** Python 3.13 / SQLAlchemy async + pgvector / Alembic / Pydantic v2 / pytest(单测 + `DOCKER_HOST` 集成)。

## Global Constraints

- 新 migration revision id `0126_memory_bitemporal`(≤32 字符);`down_revision = "0125_memory_access_count"`(当前 head)。
- 全部 7 个新列 **nullable / 非阻塞**;`valid_at` 对存量行回填 = `created_at`,其余留 NULL。
- **两个 store 全程 parity**:每个 store 方法/过滤改动必须同时落 `SqlMemoryStore`(`memory/sql.py`)与 `InMemoryMemoryStore`(`memory/memory.py`)。
- **检索默认过滤**:`retrieve()` 排除 `invalid_at IS NOT NULL`(被取代)与 `expired_at <= now()`(世界中已失效)的行;这些行 **保留在库**(历史/审计),只是不进 agent 召回。过滤只加在 `retrieve` 路径(`_vector`/`_hybrid`),不改 consolidator/purge/list 的 WHERE。
- **reconcile append-only**:UPDATE → `supersede`(关旧开新),DELETE → `expire`(设 `expired_at`)。reconcile 内不再调 `update_content`/`soft_delete`。`update_content`/`soft_delete` 本身保留(用户手动纠正/遗忘 API 仍用,不动)。
- **溯源两活站**:run-end writeback + 压缩前 flush 从 `config` 取 `run_id` 透传;**DLQ 重试路径 OUT OF SCOPE**(重试记忆 `source_run_id` 留 NULL,列为 follow-up)。
- `source_run_id` 用 **Text 列 + protocol `str | None`**,镜像既有 `source_thread_id`(它也是把 UUID 存成 Text);不用 uuid 列以免与 `source_thread_id` 混型(偏离 spec 的 "uuid",取先例一致性)。
- **不做(P5b-2 / backlog)**:时间旅行 `as_of` 检索 API、`expected_valid_days` 的主动预测复核、抽取 prompt 的 correction 标记、前端记忆 tab 跳 run/查历史、graph 关系记忆。`expected_valid_days` 列本 PR 建但不读。
- CI:mypy 范围含 `packages` + `services/orchestrator/src` + control-plane;ruff 全库 + `format --check`;pytest 含 `tools/eval`。集成测试需 `export DOCKER_HOST=unix:///Users/mac/.docker/run/docker.sock`。
- 改 `flush_messages_to_memory` 签名后,回归要跑 control-plane 测(经真 `run_agent`),非只 orchestrator。
